# Rudraksh Safar - SEO Governance & Control Framework

This document serves as the absolute operational standard for all future content expansion, ensuring strict adherence to the Elite SEO Mandate. No page shall be published without satisfying these criteria.

## 1. Content Approval System (Pre-Flight Checklist)
Before any pull request containing a new page is merged, the developer/content creator MUST answer:
- **Why does this page exist?** (E.g., To capture commercial intent for Raipur-origin flights to Kashmir).
- **What search intent does it serve?** (Navigational, Informational, Commercial, Transactional).
- **What existing page does it support?** (E.g., Supports `/domestic/kashmir`).
- **How does it contribute to authority?** (E.g., Adds unique local pricing unavailable elsewhere).
- **How does it contribute to revenue?** (E.g., Contains sticky mobile CTA for direct lead generation).

## 2. Anti-Cannibalization Matrix
To prevent internal competition for core terms like "Spiti Tour Package", we enforce a strict 1-to-1 Keyword-to-Canonical mapping.
- **Primary Pages:** Only one URL may target the broad exact-match keyword.
- **Supporting Pages:** Must target long-tail variants (e.g., "Best time for Spiti bike trip") and MUST contain an exact-match anchor link pointing back to the Primary Page.
- **Risk Assessment:** Before publishing, perform `site:rudrakshsafar.com "Target Keyword"`. If a page already ranks, update that page instead of creating a new one.

## 3. City Page Governance (The Anti-Doorway Rule)
Every localized city page (e.g., Kashmir Tour Packages from Bhilai) MUST pass the **SEO-Agnostic Test**: *Would this page still be useful if SEO did not exist?*
- Must contain **Unique Flight/Train Logistics** originating from that specific city.
- Must contain **Unique Pricing** factoring in the travel cost from that origin.
- Must contain **Local Testimonials** from residents of that city.
- **Strictly NO template duplication or automated text spinning.**

## 4. Internal Linking Governance
No page may exist in isolation.
- **Rule 1 (Parent):** Every page must link upwards to its Authority Hub.
- **Rule 2 (Commercial):** Every informational guide must link downwards/sideways to a Tier A revenue-driving package.
- **Rule 3 (Audit):** A quarterly `site:` crawl must be conducted to identify and eliminate orphan pages.

## 5. Revenue Attribution Framework (Tier System)
Budget and engineering effort must prioritize revenue.
- **Tier A (Revenue Drivers):** Core packages (e.g., Honeymoon, Bike Trips). Require max schema, CRO, and tracking.
- **Tier B (Authority Builders):** Ultimate Guides, Master Hubs. Require deep EEAT profiles and comprehensive media.
- **Tier C (Supporting Content):** FAQs, basic blog posts. Must funnel to Tier A/B.

## 6. Quarterly Authority Audit
Every 90 days, the SEO lead must compile and review:
1.  **Authority Score:** Review GSC indexed pages vs impressions.
2.  **Internal Link Score:** Run an orphan page crawl.
3.  **EEAT Score:** Verify all Author Profiles and Customer Testimonials remain accurate and verifiable.
4.  **Conversion Score:** Review GA4 `whatsapp_click` and `form_submit` metrics.

**By adhering to this framework, Rudraksh Safar will scale defensively, prioritizing predictable lead generation over vanity traffic.**
